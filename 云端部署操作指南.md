# 私密虚拟桌面 - 云端部署操作指南（小白版）

## 你需要什么

- 一台能上网的电脑
- 一个 GitHub 账号（免费注册）
- 大约 10 分钟时间

## 整体架构

```
你的浏览器
    |
    v
[ Railway（免费云服务器）] ----> [ 阿里云 MySQL 数据库（已配置好）]
    |
    +-- 前端页面（React 桌面）
    +-- 后端 API（tRPC）
    +-- 登录认证（Kimi OAuth）
```

## 第一步：把代码传到 GitHub

1. 打开 https://github.com/new 创建一个新仓库
2. 仓库名填 `private-desktop`，选择"公开"（Public）
3. 点击"Create repository"
4. 打开你的电脑终端（Windows 按 `Win+R` 输入 `cmd` 回车）
5. 依次执行下面命令（复制粘贴，每行回车）：

```bash
cd /mnt/agents/output/app
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/你的用户名/private-desktop.git
git push -u origin main
```

> 推送到 GitHub 时可能需要输入你的 GitHub 用户名和密码（或 Token）。

## 第二步：在 Railway 上部署后端

Railway 是一个免费的云服务平台，可以自动部署你的代码。

### 2.1 注册 Railway

1. 打开 https://railway.app/
2. 点击"Login"，选择"Continue with GitHub"用 GitHub 账号登录

### 2.2 创建新项目

1. 登录后点击"New Project"
2. 选择"Deploy from GitHub repo"
3. 找到并选择你的 `private-desktop` 仓库
4. 等待自动部署完成（大约 2-3 分钟）

1. 点击"Settings"标签

3. 点击"Deploy"重新部署

### 2.5 同步数据库

1. 点击项目页面的"New"按钮
2. 搜索并添加"MySQL"插件（这样 Railway 会给你一个 MySQL 数据库）

**或者**如果你已经有数据库（你当前的环境变量中已配置了阿里云 MySQL），跳过这一步。

### 2.6 获取公网地址

1. 部署完成后，点击"Settings" -> "Networking"
2. 点击"Generate Domain"，Railway 会给你一个类似 `https://your-app.up.railway.app` 的地址
3. **记下这个地址**，后面需要用到

## 第三步：配置 Kimi OAuth 回调地址

1. 打开 https://platform.kimi.com/ 登录你的开发者账号
2. 找到你的应用 `19e4ef80-b842-836e-8000-0000a899c295`
3. 在"OAuth 回调地址"中填入：
   ```
   https://你的-railway地址.up.railway.app/api/oauth/callback
   ```
   例如：`https://private-desktop.up.railway.app/api/oauth/callback`
4. 保存设置

## 第四步：更新前端代码并重新部署

现在需要让前端知道后端 API 的地址。

### 4.1 修改前端 API 地址

打开 `src/providers/trpc.tsx` 文件，将 API 地址改为你的 Railway 地址：

```typescript
// 原来（开发环境用本地地址）
const trpcClient = trpc.createClient({
  links: [
    httpBatchLink({
      url: "/api/trpc",  // <--- 改这行
      // ...
    }),
  ],
});

// 改成（生产环境用云端地址）
const trpcClient = trpc.createClient({
  links: [
    httpBatchLink({
      url: "https://你的-railway地址.up.railway.app/api/trpc",  // <--- 改成这个
      // ...
    }),
  ],
});
```

### 4.2 推送代码重新部署

```bash
cd /mnt/agents/output/app
git add .
git commit -m "update api url for production"
git push origin main
```

Railway 会自动检测代码更新并重新部署。

## 第五步：验证部署

1. 打开你的 Railway 地址（如 `https://private-desktop.up.railway.app`）
2. 你应该看到登录页面
3. 点击"本地模式"或"使用 Kimi 账号登录"测试
4. 如果能正常进入桌面，说明部署成功！

## 常见问题

### Q1: 部署后打开页面显示 404 / 空白
- 检查 Railway 的日志（点击"Deployments" -> 最新的部署 -> "View Logs"）
- 确认环境变量是否全部正确添加
- 确认 `npm start` 命令是否正确设置

### Q2: 数据库连接失败
- 检查 `DATABASE_URL` 环境变量是否正确
- 确认数据库允许外部访问（白名单/IP 限制）
- 在 Railway 日志中查看具体错误信息

### Q3: 登录后跳转回 localhost
- 检查 `VITE_KIMI_AUTH_URL` 环境变量
- 确认 Kimi 平台配置的回调地址和你的 Railway 地址一致
- 修改后需要重新部署

### Q4: 前端显示"API 不可用"
- 确认 `trpc.tsx` 中的 API 地址已改为你的 Railway 地址
- 确认 Railway 服务已正常运行
- 检查浏览器开发者工具（F12 -> Network）看请求是否发出

## 数据安全说明

- **数据库**: 你的数据存储在阿里云 MySQL 中，已通过 SSL 加密连接
- **登录**: 使用 Kimi OAuth 2.0 标准协议，不存储你的密码
- **会话**: 使用 JWT + Cookie 机制，自动过期
- **传输**: 全站 HTTPS 加密传输
